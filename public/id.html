Apa itu KAEDE?

KAEDE adalah proyek integrasi milik PT Koneksi Jaringan Indonesia yang menghubungkan Trello — tempat tim merencanakan dan melacak pekerjaan — dengan ekosistem pengembangan berbasis Agentic AI.

Dimulai sebagai Trello Power-Up untuk manajemen environment (Production / Staging / Development), KAEDE berkembang menjadi jembatan utama antara perencanaan tim di Trello dan eksekusi berbasis AI Agent melalui protokol MCP OpenCode.

1 Power-Up Trello — Kelola environment, badge, dan autorisasi langsung dari kartu Trello.
2 MCP Bridge — Membuka data Trello untuk dibaca dan ditulis oleh AI Agent melalui MCP OpenCode.
3 Knowledge Hub — Pusat informasi ekosistem Koneksi untuk seluruh anggota tim.

Protokol

OpenKB

Open Knowledge Base — bahasa komunikasi AI ↔ Human

OpenKB adalah protokol yang memungkinkan AI Agent dan manusia berbagi konteks secara dua arah. Disimpan sebagai direktori .openkb/ di setiap repositori.

Dua ruang penyimpanan:

Tracked SHARED/

Glossary istilah, referensi eksternal, keputusan arsitektur (decision-log). Komit ke git — bisa dibaca semua orang dan AI.

Gitignored PERSONAL/

Catatan pribadi, preferensi lokal, eksperimen. Khusus individu — tidak masuk git, tidak terbaca orang lain.

Struktur Direktori

# Setiap repo bisa punya ini:
.openkb/
├── SHARED/
│   ├── glossary.md      # Istilah & singkatan
│   ├── references.md    # Link & referensi
│   ├── decision-log.md  # Keputusan arsitektur
│   └── project-context.md # Konteks proyek
└── PERSONAL/
     └── .gitignore      # Tidak di-track

Alur Komunikasi

→ AI membaca OpenKB sebelum membantu — paham konteks, istilah, aturan.
→ Manusia menulis OpenKB — catat keputusan, update glossary, tambah referensi.
→ Dua arah — AI bisa rekomen update ke OpenKB, manusia review dan approve.

Konfigurasi

OpenCode

Konfigurasi AI Agent di setiap repositori

OpenCode adalah tools AI Agent yang digunakan tim Koneksi. Setiap repositori bisa memiliki direktori .opencode/ yang mengkonfigurasi bagaimana AI Agent berperilaku di repo tersebut.

Agent Types

Setiap repo punya beberapa agent dengan peran berbeda: build (eksekusi), plan (perencanaan), review (review), docs (dokumentasi). Masing-masing punya permission berbeda.

Project Context

File SHARED/project-context.md dan SHARED/agent-rules.md memberi tahu AI tentang stack teknis, tim, aturan main, dan kebiasaan repo tersebut.

Contoh Struktur

.opencode/
├── opencode.json       # 4 agent types + permissions
└── SHARED/
     ├── project-context.md # Stack, tim, repo
     └── agent-rules.md    # Aturan main agent

Integrasi dengan KAEDE

Melalui MCP (Model Context Protocol), OpenCode bisa memanggil KAEDE untuk membaca dan menulis data Trello — membuat kartu, mengubah label, mengecek assignment — langsung dari percakapan dengan AI Agent.

Workflow

Team Agentic Coding Operation

Koneksi menjalankan pengembangan dengan pendekatan Human In The Loop (HITL) — AI Agent membantu eksekusi, manusia tetap memegang kendali.

1

Perencanaan Product + BA

Product Manager dan Business Analyst menyusun kebutuhan di Trello: membuat kartu, menentukan prioritas, melampirkan konteks. Setiap kartu bisa diberi label environment (Production/Staging/Development) melalui KAEDE Power-Up.

2

Konteks AI OpenKB

Sebelum memulai, AI Agent membaca OpenKB dan OpenCode di repositori untuk memahami konteks: istilah proyek, keputusan arsitektur, stack teknis, aturan main. Agent juga bisa membaca kartu Trello terkait melalui KAEDE.

3

Eksekusi Developer + Agent

Developer (Frontend, Backend, UI/UX, Junior) bekerja dengan bantuan AI Agent. Agent membantu menulis kode, mengecek pattern, menjalankan test. Developer tetap memegang kendali — AI adalah pair programmer, bukan pengganti.

4

Review Senior (HITL)

Human In The Loop — Senior Developer me-review semua perubahan sebelum di-merge. Memastikan kualitas, konsistensi, dan keamanan. AI Agent membantu review (static analysis, pattern check) tapi keputusan akhir tetap di tangan manusia.

5

Deploy & Laporkan CI/CD

Setelah di-merge, otomatis terdeploy. Kartu Trello diupdate — status, environment, link deploy. Semua terdokumentasi di decision-log OpenKB untuk konteks masa depan.

Panduan

Menggunakan KAEDE Power-Up

Cara menginstall dan memanfaatkan KAEDE di Trello tim Koneksi.

1

Install Power-Up

Buka trello.com/power-ups/admin, klik Create atau tambahkan KAEDE yang sudah didaftarkan. Masukkan URL: https://kaede-powerup.netlify.app

2

Aktifkan di Board

Buka board Trello tim → Power-Ups → cari "KAEDE" → Enable. Tombol KAEDE akan muncul di header board.

3

Kelola Environment

Buka kartu → klik KAEDE: Environment → pilih Production / Staging / Development. Badge akan muncul di muka kartu.
